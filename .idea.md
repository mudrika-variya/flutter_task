# Flutter Technical Task

## **Mini Tournament, Wallet & Authentication Module**

**Time Limit: 3 Hours**

Objective:

Build a small Flutter module that simulates a tournament joining flow for a competitive esports platform. This task evaluates architecture, state management, business logic handling, authentication handling, UI responsiveness, and clean coding practices.

### **1. Authentication Screen (Login)**

- Email field
- Password field
- Validate email format
- Password minimum 6 characters
- Use https://dummyjson.com/docs/auth for Login (dummy auth API)
- Save mock token locally
- Show loading indicator
- Handle invalid credentials

> **Implementation note:** for testing the app uses hardcoded credentials (`username: 'emilys'`, `password: 'emilyspass'`) and pretends to hit the dummy API.


### **2. Tournament List Screen**

- Tournament Name
- Entry Fee
- Prize Pool
- Total Slots
- Available Slots
- Join Button
- Display logged-in email
- Display wallet balance
- Include Logout button

### **3. Wallet Balance**

- Initial Wallet Balance: ₹500

### **4. Join Tournament Logic**

- If wallet < entry fee → show error
- If slots = 0 → show 'Tournament Full'
- Deduct entry fee
- Reduce available slots
- Disable join button after joining
- Prevent double joining

### **5. Leaderboard Screen**

- Rank
- Player Name
- Points
- Minimum 20 players (add pagination)
- Implement pagination or lazy loading
- Smooth scrolling
- Handle loading state

### **Technical Requirements**

- Use proper state management (Provider / GetX / Bloc)
- Follow clean architecture principles
- Separate UI from business logic
- Handle loading, error, empty states
- Use null safety properly

### **Bonus (Optional)**

- Secure token persistence
- Pull to refresh
- Const constructors
- Model equality
- Unit test for wallet logic
- Route guard for authentication

### **Submission Requirements**

- Source code (GitHub or ZIP)
- README explaining architecture decisions and state management choice

### **Evaluation Criteria**

- Architecture & structure
- Authentication handling
- State management quality
- Business logic correctness
- Edge case handling
- Code readability
